# 需求说明

## 背景

本项目用于构建一个本地自用的 RAG 系统。系统接收文档资料，将资料转为可检索知识库，并在用户提问时结合检索结果生成回答。

## 核心用户

- 使用者：本地运行服务、配置模型、管理数据源。
- 知识库维护者：上传文档、触发索引、查看处理状态。
- 终端用户：提出问题，获得可追溯来源的回答。

## 功能需求

### 文档管理

- 上传 PDF、DOCX、TXT、Markdown 文件。
- 记录文件名、大小、类型、上传时间、处理状态。
- 支持重新索引文档。
- 支持删除文档及其关联切片和向量。

### 文档处理

- 提取原始文本。
- PDF 默认使用 Docling 做 layout-aware 解析，优化论文双栏、表格、图注和阅读顺序。
- 对文本进行清洗和分段。
- 将分段结果保存为 chunk。
- 调用 embedding provider 生成向量。
- 将向量写入向量数据库。
- 记录处理日志和错误信息。

### 检索问答

- 接收用户问题。
- 对问题生成 query embedding。
- 从向量数据库检索相关 chunk。
- 可选 rerank。
- 将检索片段组装为 prompt。
- 调用 LLM 生成回答。
- 返回回答、引用来源、命中文档和 chunk 信息。

### 管理能力

- 查看文档处理状态。
- 查看知识库统计信息。
- 查看最近问答记录。
- 配置检索参数，如 top_k、score_threshold、max_context_tokens。

## 非功能需求

- 服务可以在本地开发机直接运行。
- 模型 API key 不写死在代码中，统一从环境变量读取。
- 单个文档处理失败不能影响整个服务。
- API 返回结构稳定，便于未来接前端。
- 关键流程有日志。
- 业务代码和 provider 实现解耦。
- 支持本地 Ollama 和云端大模型 API 两种模型模式。

## MVP 范围

第一版只实现单知识库，并优先保证本地功能可用：

- 文件上传
- 文档解析
- 文本切分
- 向量化
- 检索问答
- 引用返回
- 本地命令行启动

暂不实现：

- 多租户
- 权限系统
- 复杂工作流
- 高级 reranker
- Web 前端
- Docker 部署
- 异步任务队列
